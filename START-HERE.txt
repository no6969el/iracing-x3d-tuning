iRacing X3D Tuning Kit  -  v2.2.0
============================================================
This is the complete kit. Nothing to merge, nothing to configure.

HOW TO INSTALL
  1. If you have an older copy, rename that folder to "kit-OLD"
     (keep it until you've confirmed this one works).
  2. Put this folder in its place.
  3. Double-click Start-Tuning-Menu.bat

  Do NOT copy individual files across from an older kit. The scripts
  now share scripts\X3D-Profiles.ps1, so an old copy of one script
  mixed in with new ones will produce wrong core numbers.

REQUIREMENTS
  Windows PowerShell 5.1 - built into Windows 10 and 11.
  PowerShell 7 is NOT required for any part of this kit.
  (Test-UndervoltStability can optionally use PS7 for AVX-512
  coverage, but works fine without it.)

  Process Lasso (free, bitsum.com) if you have a dual-CCD chip.
  The menu tells you whether you need it.

FIRST RUN
  Your CPU is detected automatically. Any saved profile from an older
  kit is ignored, because the format changed.

  Check the header shows the right chip. If it doesn't:
     Tuning-Menu > CPU profile > pick your chip class

SUPPORTED PROCESSORS
  6-core single-CCD   5500X3D  5600X3D  7500X3D  7600X3D
  8-core single-CCD   5700X3D  5800X3D  7700X3D  7800X3D  9800X3D  9850X3D
  12-core dual-CCD    7900X3D  9900X3D
  16-core dual-CCD    7950X3D  9950X3D
  16-core, both cached  9950X3D2 Dual Edition
  mobile              7945HX3D  9955HX3D

  Not an X3D? The kit still runs. The general fixes all apply; core
  pinning and interrupt steering are skipped automatically.

FOLDER LAYOUT  -  keep this structure intact
  Start-Tuning-Menu.bat        <- start here
  Apply-Baseline.bat           <- or here, for the one-shot version
  Tuning-Menu.ps1
  Apply-Baseline.ps1
  scripts\                     <- all the individual tools
      X3D-Profiles.ps1         <- shared CPU detection, don't delete
      README.txt               <- what every script does
  tests\                       <- development only, safe to delete
  RELEASE-NOTES.md             <- what changed in this version
  CHANGELOG.md                 <- full project history

IMPORTANT  -  the per-session routine
  Pre-Race-Quiet disables the Windows Update scan TASKS, stops the
  update/search services, and turns off Defender real-time for the
  session.

  The stopped services and Defender come back on their own (and always
  after a reboot). The disabled scan tasks do NOT - they stay off until
  Post-Race-Restore re-enables them.

  So run Post-Race-Restore after every session. Check-Quiet-Status
  shows which state you're in.

IF WINDOWS BLOCKS THE SCRIPTS
  The .bat launchers already use -ExecutionPolicy Bypass, so this is
  rare. If SmartScreen blocks a .bat after downloading, right-click
  the ZIP -> Properties -> tick Unblock, THEN extract. Doing it on the
  zip handles every file at once.

WHAT CHANGED IN v2.2.0
  Every X3D AMD has shipped is now supported, including the 6-core
  parts (5500X3D, 5600X3D, 7500X3D, 7600X3D) that had no option at all
  before, plus the 7700X3D, 9850X3D, the dual-cached 9950X3D2, and the
  mobile HX3D chips. Non-X3D CPUs are supported for the general fixes.

  Fixed a real bug: on a 6-core chip the old code aimed interrupts at
  CPU 8 and CPUs 9-11. With SMT off those don't exist, so it wrote
  affinity masks pointing at nothing. Targets are now validated against
  the CPUs Windows actually reports and can never be out of range.

  Full detail in RELEASE-NOTES.md.

TESTING (optional)
  You can make the whole kit behave as any supported chip:

     $env:X3D_FORCE_PROFILE = '5600X3D'
     .\Start-Tuning-Menu.bat

  Anything that writes to the registry switches to DRY RUN and changes
  nothing. Close the window to go back to normal.

  tests\test-integration.ps1 sweeps all 17 chips and checks every core
  target is valid:

     powershell -ExecutionPolicy Bypass -File .\tests\test-integration.ps1

  It should end with "RESULT: all assertions passed".

These scripts change Windows/registry/power settings. Every one has an
Undo or is reversible, but review before running on someone else's PC.
