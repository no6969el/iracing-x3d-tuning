The one thing to understand: your two CCDs

A dual-CCD X3D chip is two core clusters:

  • CCD0 — the V-Cache die. Huge L3 cache, slightly lower clocks. Games love this die — cache beats MHz in a sim.
  • CCD1 — the frequency die. Higher clocks, normal cache.

The whole game: keep the sim on the V-Cache die, keep everything else off it.

CPUV-Cache CCD (sim here)Frequency CCD (everything else)
9950X3D / 7950X3D (16-core)logical cores 0–15logical cores 16–31
9900X3D / 7900X3D (12-core)logical cores 0–11logical cores 12–23

Wherever this guide says “core 16” (the first frequency-CCD core), a 12-core owner uses “core 12.” Every script has this as an editable value at the top.

Golden rules

Learned the hard way

EAC blocks affinity

EasyAntiCheat rejects external CPU-affinity changes to iRacing — hard-pinning reverts to all cores seconds after launch. Use a soft CPU Set instead. The mask reading 0xFFFFFFFF is normal for CPU Sets, not a failure.

Unparked plan, NOT Balanced

Single-CCD guides say “Balanced.” On dual-CCD with VR, Balanced’s core parking strands the VR compositor and starves the GPU. Keep all cores unparked.

Measure, don’t guess

Frametime hiccups rarely show without instrumentation. Log first, change one thing, log again.

The VR focus/timer trap

Windows 11 only honors a game’s high-res timer while it owns foreground focus — and in VR, the compositor steals focus constantly. If your stutter stops when you click the sim window, that’s the signature. Fix #3 ends it.

Toolkit

Measure first

  • FullTrace.ps1 — main logger. ~1/sec CSV of power plan, per-CCD load, CPU-0 vs frequency-die interrupt/DPC, GPU util/power/throttle, sim & VR CPU, pagefaults. Gaps in the timestamps = a stall caught live.
  • Preflight-Check.ps1 — confirms every fix is live before a session.
  • LatencyMon — names the driver behind any DPC spike. Essential for the last mile.
  • Repair-PerfCounters.ps1 — if per-core columns come back empty, your Windows performance counters are broken. Rebuild them.
  • Watch-TimerResolution.ps1 — live view of the system timer resolution + which window has focus. Catches the “stutters until I click” focus/timer problem in the act (see fix #3).

Good result = CCD0 busier than CCD1, GPU util steady with no collapses, no time-gaps, hard pagefaults near zero (big bursts during loading are normal).

The fixes

Highest impact first

1Power plan: all cores unparked (NOT Balanced). Bitsum Highest Performance or Windows High Performance. If using ParkControl: Parking = Off, and disable “Dynamic Boost” — it flips the plan mid-race and causes hard freezes (our worst offender).

2Process Lasso: pin the sim to the V-Cache die. Add a CPU Set for iRacingSim64DX11.exe = cores 0–15 (0–11 on 12-core). Put VR/overlays/wheel/browsers on the frequency die (16–31). Exclude iRacing from ProBalance. Optional: High I/O, memory, and CPU priority.

3Global timer resolution — the “stutters until I click” fix (Windows 11, big for VR). Windows 11 drops the system timer from ~1 ms to 15.625 ms whenever the sim window loses foreground focus — which the VR compositor causes constantly — producing periodic frame-pacing hitches that stop the moment you click the sim window. Confirm it with Watch-TimerResolution.ps1, then run Enable-GlobalTimerResolution.ps1 (admin) → reboot. It restores the pre-Win11 behavior: high-res timer requests honored globally, focus or not. Reversible with Undo-GlobalTimerResolution.ps1.

4GPU interrupt affinity off CPU 0. The sim thread favors CPU 0 and so do NVIDIA’s GPU interrupts — they collide. Set-GPU-IRQ-Affinity.ps1 (admin) steers them to the first frequency-die core (CPU 16, or 12 on a 12-core) → reboot → verify in LatencyMon that nvlddmkm.sys DPCs left CPU 0.

5NIC + USB interrupt affinity. Optional. Set-NIC-USB-IRQ-Affinity.ps1 steers network + USB interrupts to frequency-die cores. Worth it only if a hitch remains after the fixes above — network DPC is usually small.

6Repair performance counters if traces show empty per-core data. Repair-PerfCounters.ps1 (admin) → reboot.

7Quiet background + Defender. Pre-Race-Quiet.ps1 disables Windows Update/Orchestrator/Search scans (they fire every ~5–7 min and cause periodic stalls) and optionally Defender real-time for the session (needs Tamper Protection off). Post-Race-Restore.ps1 reverts. Add-Defender-Exclusions.ps1 excludes iRacing’s folders so Defender stops scanning texture reads mid-race.

8HAGS / Game Bar / USB Selective Suspend. Apply-Guide-Extras.ps1 turns off USB Selective Suspend (stops wheel/VR USB power-cycling) and Game Mode/Bar/DVR. Test HAGS both ways — it’s a toss-up on newer cards.

9iRacing in-game settings (edit with the sim CLOSED): LoadTexturesWhenDriving=0 (preload instead of streaming mid-lap), CacheSwap3HighResCars=0, spoolingEnabled=0 in app.ini (no continuous replay-to-disk), CrowdDetail=0.

10NVIDIA Control Panel (per-app iRacingSim64DX11.exe): Low Latency = On (not Ultra), Power = Prefer Max Performance, V-Sync Off, Threaded Optimization On, Shader Cache Unlimited, Anisotropic 16x. Disable the in-game overlay.

11Remove monitoring overlays. RTSS / MSI Afterburner are notorious DPC + hard-pagefault sources. Close them, especially while diagnosing.

VR

Clarity without hiccups

  • Supersampling beats MSAA for clarity. MSAA only smooths polygon edges; it does nothing for fences, distant lines, and specular shimmer. Resolution anti-aliases everything. Keep MSAA 4x (8x often crashes iRacing in VR) and spend headroom on resolution.
  • Set resolution in ONE place. Runtime panel or in-sim slider — not both (they multiply and you’ll miss frame deadlines).
  • Quad-views / foveated is a big free win — sharp center, low-res periphery. When the runtime provides it (e.g. Pimax’s Advanced Quad-View), the runtime panel is authoritative; the sim just renders into it.
  • Sharpening amplifies shimmer — lower it and let resolution do the work.
  • Keep ~15% GPU headroom (util peaks under ~85%). VR punishes any missed frame with reprojection judder.
  • Stutter that stops when you click the sim window? That’s the focus/timer problem — see fix #3. It hits VR hardest because the compositor keeps taking foreground focus from the sim.
Why this differs

Adapted from a single-CCD guide — two things flip

This method builds on an excellent single-CCD guide (rcsracing93.github.io/iracing-stutter-fix). Most transfers, but on dual-CCD:

  1. Power plan: single-CCD says Balanced; dual-CCD with VR needs all cores unparked (Balanced parks cores and starves the VR compositor).
  2. CCD pinning matters — single-CCD has nothing to pin. Getting the sim onto the V-Cache die and everything else off it is half the battle.
Before you run anything: these scripts change Windows / registry / power settings. Every one is reversible (an Undo script or a documented toggle), but review before running — it’s your PC, at your own risk.
