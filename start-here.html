Start Here — Guided Walkthrough

A step-by-step version that walks you through the whole process, assuming you've never run a PowerShell script. No technical knowledge needed — just follow the steps in order.

🖱️ Easiest of all: use the interactive menu

After you unzip the kit, just double-click Start-Tuning-Menu.bat. It opens a clean menu that runs every step for you in its own window, detects your CPU/GPU on first launch (and remembers it), sets the right core numbers automatically, and shows you what's happening the whole time.

If you use the menu, you can skip the manual steps below — they're here for reference, or for anyone who prefers doing it by hand.

⚠️ These scripts change Windows settings (power, registry, services, Defender). Every one is reversible — there's an Undo for anything that changes a setting. Nothing here is dangerous if you follow the order. Go slowly and don't skip the reboots.
Setup

Before you start (3 minutes)

  1. Download the kit — the .zip, or the green Code ▸ Download ZIP on the repo.
  2. Unzip it to a permanent folder — e.g. C:\iRacingTuning\. Not your Downloads or a temp folder, or the shortcuts break later.
  3. Know your CPU's core split (you'll use it once):
    Your CPUV-Cache cores (sim here)Frequency cores (everything else)
    9950X3D / 7950X3D0–1516–31
    9900X3D / 7900X3D0–1112–23
    12-core owners change one number in one script — the checklist tells you when.
  4. Install Process Lasso (free version is fine, from bitsum.com) — a third-party app used for one step.
Read once

Part 1 — How to run these scripts

A PowerShell script is a .ps1 file. Double-clicking one does NOT run it — Windows opens it in an editor. So we run them one of two ways.

✅ The easy way: make double-click shortcuts (do this first!)

The kit includes a helper, Create-Launchers.ps1. Its only job is to create a clickable shortcut next to every script, so from then on you just double-click. The shortcuts are pre-configured — the ones needing admin rights ask permission automatically, and the logging ones stay open so you can read results.

Run it once:
  1. Open your scripts folder.
  2. Right-click Create-Launchers.ps1 → “Run with PowerShell.”
  3. If asked about an execution policy, type Y and press Enter.
  4. Done — a .lnk shortcut now sits next to each script. From now on, just double-click the shortcut you want.

The manual way (backup method)

  1. Click Start, type PowerShell.
  2. Normal script: click Windows PowerShell. Admin script: right-click → “Run as administrator” → Yes. (The title bar says Administrator — that's how you know.)
  3. Type this, then drag the .ps1 onto the window (it pastes the path), and press Enter: 
    powershell -ExecutionPolicy Bypass -File "PASTE THE FILE PATH HERE"
    (-ExecutionPolicy Bypass just lets this one script run; it changes no Windows setting.)

Which scripts need admin? Any that change a setting — marked ADMIN below. The double-click shortcuts handle this for you.

The checklist

Part 2 — Step by step, in order

Tick each box as you go. Don't rush the reboots — some changes only apply after a restart.

Phase 1 — Set up & get a “before” picture

  • Unzip the kit to a permanent folder (e.g. C:\iRacingTuning\).
  • Run Create-Launchers.ps1 once (Part 1) so everything becomes double-click.
  • Double-click FullTrace, race ~10 min, press Ctrl+C. Saves a CSV to your Desktop — your “before” measurement. (Optional but recommended.)

Phase 2 — Fix the foundation

  • ADMIN Run Repair-PerfCounters only if FullTrace's per-core columns were blank. Then reboot.
  • Set the power plan to all cores unparked — Bitsum Highest Performance or Windows High Performance. If using ParkControl: Parking = Off, uncheck “Dynamic Boost.”
  • In Process Lasso, right-click iRacingSim64DX11.exe → CPU Sets → your V-Cache cores (0–15, or 0–11). Also exclude it from ProBalance.
  • 12-core owners only: open Set-GPU-IRQ-Affinity.ps1 in Notepad, change $TargetCore = 16 to = 12, save.
  • ADMIN Run Set-GPU-IRQ-Affinity. Then reboot.

Phase 3 — Quiet the system (each once)

  • ADMIN Run Add-Defender-Exclusions.
  • ADMIN Run Apply-Guide-Extras.
  • Adjust iRacing's in-game settings (with iRacing fully closed): texture preload on, replay spooling off, crowds off.
  • Set the NVIDIA Control Panel options for iRacing (see full guide, fix #9).

Phase 4 — Every race

  • ADMIN Run Pre-Race-Quiet before racing.
  • Race. 🏁
  • ADMIN Run Post-Race-Restore after — turns Windows Update, Search & Defender back on. Don't skip it.

Phase 5 — Prove it worked

  • Run Preflight-Check — prints a green READY if everything's set.
  • Run FullTrace for a race, Ctrl+C, and compare the CSV to your “before.”
Verify

Part 3 — How to tell it worked

Open the FullTrace CSV (Excel or Notepad). You want:

  • No skipped seconds in the timestamp column — a jump like 12:04:40 → 12:04:42 means a stall. Fewer/none = better.
  • The power_plan column stayed on your unparked plan the whole time.
  • The gpu_util column stays steady while racing (no sudden drops to single digits).

Smooth on track + no timestamp gaps = you're done. 🎉

Safety net

Part 4 — Undo anything

Nothing is permanent. To reverse a change, run its Undo ADMIN, then reboot:

  • Undo-GPU-IRQ-Affinity · Undo-NIC-USB-IRQ-Affinity · Undo-Guide-Extras
  • Post-Race-Restore — turns Windows Update/Search/Defender back on
  • Power plan / Process Lasso / iRacing settings — just set them back in their apps
Plain English

Part 5 — Glossary

  • CCD — one of the two core clusters on your CPU. One has extra cache (V-Cache), one clocks higher.
  • CPU Set — a gentle “please run here” hint that steers a program onto certain cores.
  • DPC / interrupt — tiny background jobs the CPU handles for your hardware. Too many on the sim's core = micro-stutters.
  • Hard pagefault — the PC pausing to read from disk. Normal during loading, not while racing.
  • Reboot — restart the PC. A few changes only apply after one.
